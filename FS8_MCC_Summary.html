主要功能

Main Functions — MPSS 四大功能模組
Module 1 · Planning
資源規劃 Planning Application
排程作業必須的前置作業
  • 資源定義（衛星、地面站、海外站）
  • 衛星記憶體 / 電量參數設定
  • 地面站維護週期與可用性配置
  • RTS 海外站排程請求與管理
  • 事件模板管理（9 種 .etf 類型）
Output：MPSS DB 資源組態、海外站請求檔
Module 2 · Scheduling
排程規劃 Scheduling Application
排程核心，7 個 CSU 組成
  • Session Manager：定義排程時間區段
  • Cold Scheduler：自動預先排程（健康觀測活動）
  • External Activity Scheduler：RSI/科學酬載排程匯入
  • Activity Request Editor：手動建立衛星活動
  • Gantt Chart：圖形化排程調整介面
  • Scheduler Validation：10 項規則驗證
  • Schedule Report：ASR / Duty Cycle / SSR utilization
Output：無衝突排程結果、作業報表
Module 3 · Product Generation
指令集產生 Product Generation
根據驗證後排程結果產生衛星指令
  • 依事件模板 (Event Template) 展開指令時序
  • 產生 TTQ（Time-Tagged Queue）
  • 產生 MPQ（Mission Planning Queue）
  • 產生 GEO Command（FS8 新增，地理指令）
  • 產生 Pass Plan（地面站接觸計畫）
Output：指令佇列檔 → MOC ARCOS → 衛星
Module 4 · Utilities
輔助工具 Utilities
4 個 CSU 組成
  • Data Transfer：SFTP 下載軌道資料、Mission Timeline
  • Product Transfer：SFTP 上傳指令集至 ARCOS
  • Purge Database/File：定期清除過期暫存資料
  • Database Management：定時備份 *.dump，同步至備援主機
Output：DB 備份檔、傳輸確認
FS8 支援衛星活動類型（8 種）
RSI Record RSI Playback Attitude Maneuver Orbit Maneuver Go Home Maneuver Misc Cmd Procedure SSR Delete Monitor

技術重點摘要

Technical Highlights
自主研發系統
MPSS 由 TASA 操控組自主研發，已實際應用於福衛五號及福衛七號任務操作。福衛八號版本為系統升級，具完整的自主維護與演進能力。
Web-Based 操控
使用者介面以 HTML + JavaScript 實作，操作人員可透過任何瀏覽器、任何具安全認證網路的 PC、NB 或手持裝置存取 MPSS，不侷限於 SOCC 內部。
3-Tier 開放架構
採用 Java 應用伺服器 + PostgreSQL（Open Source） + XML 通用交換格式，符合開放標準，具備跨平台性與未來擴充性。
FS8 新增功能
針對 FS8 新需求客製化建置：轉動衛星進行資料下載、GEO Command 封裝（地理位置觸發指令）。
高可用性備援
Primary / Backup 雙主機架構，每日自動同步 PostgreSQL 資料庫（*.dump），確保系統在硬體或 OS 崩潰時可立即切換，不中斷任務操作。
排程驗證機制
Scheduler Validation CSU 內建 10 項規則驗證（含 Duty Cycle、Maneuver Interval、Contact Bounds、RFPA Window 等），確保排程結果無衝突、符合衛星操作限制。
面向 福衛五號 / 七號 (基礎) 福衛八號 (升級重點)
指令類型 TTQ、MPQ TTQ、MPQ、GEO Command（新增）
取像模式 標準影像取像 TDI 影像取像（新增）
資料下載 固定姿態下載 轉動衛星進行資料下載（新增）
瀏覽器相容 需限制 Chrome 版本 修正 Gantt Chart，不再限制 Chrome 版本
記憶體需求 N/A 建議 32GB 以上（系統無硬性限制）
已驗證衛星 FS5（遙測）、FS7（氣象） 以 FS5 為基礎，融入多年操作經驗優化

Satellite & Ground Resource Management
衛星資源 Satellite Resources
  • 記憶體 (SSR)：固態紀錄器容量配置與使用追蹤
  • 電力 (Power)：衛星電量消耗參數與 Duty Cycle 管理
  • 姿態限制：各活動之姿態操控時間與間距限制
  • 指令佇列：TTQ/MPQ/GEO 指令序列時序管理

地面站資源 Ground Station Resources
  • S 頻段站（中壢/台南）：0° / 10° 仰角接觸視窗
  • X 頻段站：11.5° 仰角接觸視窗（影像下載）
  • 維護週期：地面站不可用時段定義
  • Turnaround Time：同站兩次接觸最小間隔
  • Contact Bounds：接觸時間上下限設定

衛星活動類型與限制條件

Satellite Activity Types & Scheduling Constraints
FS8 衛星活動類型（8 種）
# 活動類型 英文名稱 來源 說明
1 RSI 取像 RSI Record IPS-PSS Mission Timeline 遙測影像感測器 (RSI) 對地取像，佔用 SSR 記憶體與電力，需在 RFPA 視窗內排程
2 RSI 下載 RSI Playback 排程系統自動產生 將 SSR 中的 RSI 影像透過 X-Band 站下傳，與 RSI Record 不得重疊
3 姿態機動 Attitude Maneuver 手動排程 / 外部請求 FS8 新增：轉動衛星以最佳化取像角度或配合下載活動；受 Latitude Time / Maneuver Interval 限制
4 軌道機動 Orbit Maneuver FDF 軌道規劃 軌道維持或轉換，消耗推進燃料，需與其他活動保持安全間隔
5 Go Home 機動 Go Home Maneuver 手動排程 衛星回歸標準姿態，受 Latitude Time Go Maneuver 限制
6 指令程序 Misc Cmd Procedure 手動 / 冷排程 例行維護指令序列、特殊操作程序，格式為 RTCMD 或 MISC 指令封包
7 SSR 刪除 SSR Delete 排程後自動產生 清除 SSR 已下傳資料以釋放記憶體空間，供後續取像活動使用
8 健康狀態觀測 Monitor Cold Scheduler 自動產生 每個 Session 自動排入，透過 S-Band 站接觸期間監視衛星健康狀態
Scheduler Validation — 10 項排程限制條件 （任一違反則排程驗證失敗，需調整後重新驗證）
R1
RSI cannot overlap PBK
RSI 取像活動與任何 Playback（影像下載）活動不得在時間上重疊，防止 SSR 同時讀寫衝突。
R2
Site Turnaround Time
同一地面站兩次連續接觸之間，必須保留最小間隔時間（Turnaround），避免天線設備來不及重置。
R3
Antenna Unavailable for Time Range
當地面站標記為維護或不可用時，排程系統不得在該時段安排任何接觸活動。
R4
Contact Bounds
接觸活動必須排在地面站的可視仰角時窗內（S-Band 0°/10°、X-Band 11.5°、RTS 30°），超出邊界則無效。
R5
Duty Cycle
RSI 取像或其他高耗能活動在特定軌道圈次內的總工作比例，不得超過衛星電力預算所允許的 Duty Cycle 上限。
R6
Rec RSI Capacity
RSI 取像累積資料量不得超過 SSR 剩餘可用容量，防止固態紀錄器溢出導致資料遺失。
R7
Latitude Time Maneuver
衛星轉動（Attitude Maneuver）只能在特定緯度時間範圍內執行，避免機動期間通過特定地區影響操控。
R8
Latitude Time Go Maneuver
Go Home 機動（回歸標準姿態）同樣受緯度時間限制，必須在允許的緯度帶內完成機動動作。
R9
Maneuver Interval
兩次姿態或軌道機動之間必須保持最小時間間隔，確保衛星姿控系統有足夠時間穩定。
R10
RSI Record FPA Window
RSI 取像活動必須排在 FPA 時窗內，FPA 開機需等待 25 秒，關機需等待 8 秒，超出保護視窗外的排程將被驗證拒絕。
