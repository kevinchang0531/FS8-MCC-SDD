MCC 系統定位與角色

System Context

任務控制中心 (Mission Control Center, MCC) 為福衛八號衛星操控中心 (SOCC) 內的次系統，提供福衛系列衛星任務操作的支援。主要工作為負責規劃衛星活動與管理地面站資源，以及進行衛星酬載儀器的科學實驗，產生沒有衝突的排程結果。並根據排程結果產生每日的指令序列以供衛星操作員上傳至衛星，衛星便會依照絕對時間執行事先安排好的動作。

MOC

任務操作中心

FDF

Ground Comm. Network

MCC 可將空中及地面作一併考量，進行衛星及地面資源及限制的統合，依照酬載儀器的任務目標，將有限的資源作有效運用，並產生無衝突且具最佳化排程的衛星與酬載指令集，此指令集傳送到 MOC 進行上傳，衛星將依據指令的時序執行各項規劃活動，達成任務目標。

MCC Subsystem Design
3.3.1 概述 MCC Overview

任務控制中心 (Mission Control Center) 為 SOCC 內的次系統，提供福衛系列衛星任務操作的支援，主要工作為負責規劃衛星活動與管理地面站資源以及進行衛星酬載儀器的科學實驗，產生沒有衝突的排程結果。並根據排程結果產生每日的指令序列以供衛星操作員上傳至衛星，如此一來衛星便會依照絕對時間執行事先安排好的動作。

3.3.3 次系統架構 MCC Subsystem Architecture

MCC 系統架構主要包含五項主要功能 CSC（Computer Software Component），每個 CSC 再由許多 CSU 所組成：

① 軌道事件處理 CSC
Orbit Event Process

處理從 FDF 取得的軌道事件資料，存入資料庫供排程規劃使用。

② 資源管理 CSC
Resource Management

提供管理地面站與衛星資源功能，以利於進行預先規劃。

③ 排程事件規劃 CSC
Scheduling / Rescheduling

提供 Gantt Chart 等輔助工具，進行酬載任務排程。

④ 衛星指令集產生 CSC
Product Generation

根據規劃的事件與指令資料庫產生衛星指令集與指令時間序列報告。

⑤ 輔助工具 CSC
Utilities

提供指令與排程報告傳遞功能，資料清除與資料庫管理。

3.3.4 次系統設計描述 MCC Subsystem Design

任務排程規劃系統 (Mission Planning and Scheduling System, MPSS) 為衛星操控中心任務控制小組所使用的排程軟體，主要負責規劃衛星所使用的地面站資源，並依需求進行衛星任務活動的排程作業，以產生適當的衛星指令。該系統以內建的排程演算法進行各項資源的最有效運用，同時積極避免資源衝突的發生。

操控組自主發展的排程系統已實際應用於福衛五號及福衛七號的任務操作，福衛八號將以福衛五號排程系統為基礎，結合多年操作經驗進行優化與調整，並針對 Video mode 取像、衛星轉動下載、GEO Command 封裝、信息傳遞等新功能進行客製化建置。
3.3.4.1 系統操作硬體環境 Hardware Architecture

為確保 MPSS 系統可持續運行，建議建立兩套排程系統 (Primary/Backup) 互為備援，每天系統會自動同步資料，讓資料庫內的資料都能保持最新狀態。

MCC 次系統對於 MOC、FDF、SCC 等次系統之相關介面流程請參考 MCC 資料流相關屬性表。資料流分為 Receiving 與 Sending 兩個方向（表 3.3.4.2-1 / 2）。

3.3.4.3 設計描述 Design Description — MPSS

MPSS 採用三層式 (3-Tiered) 架構的應用程式：

終端機層 (Client)

使用者介面以 HTML + JavaScript 撰寫，使用者可用一般瀏覽器即可進行操作，不侷限於 SOCC 內。

應用伺服器層 (App Server)

採用 Java 程式發展，跨平台語言，採用通用交換格式 XML。

資料庫層 (Database)

採用 Open Source PostgreSQL，整體執行環境為 Windows 平台。

MPSS 包含四大主要功能：

# 功能模組 說明
1 Planning Application 資源規劃：資源定義、參數設定、資源配置（排程前置作業）
2 Scheduling Application 排程核心：預先排程、特定酬載排程、手動排程調整與排程檢驗，並依排程結果產生對應報表
3 Product Generation 依據排程結果產生正確的衛星指令，由操作人員上傳給衛星據以執行排定的衛星活動
4 Utilities 提供其他功能輔助排程系統作業之進行

3.3.4.4 設計限制 Design Constraints
  • 資源管理 CSC 必須提供 user-friendly 操作介面，讓操作人員可彈性定義電源管理、衛星記憶體配置、地面站維護週期與海外地面站可用性等參數。
  • RTS 排程請求/確認時間表必須遵循指定格式，參考 Remote Tracking Station ICD (FS8-ICD-0003)。
  • RSI 取像、衛星轉動等活動請求必須包含在任務時間表 (Mission Timeline) 中，格式定義於 Ground Segments ICD (FS8-ICD-0001)。
  • 排程結果必須符合操作規則與資源限制條件（定義於 Rule Document [TBD]）。
  • 福衛八號活動類型：RSI Record RSI Playback Attitude Maneuver Orbit Maneuver Go Home Maneuver Misc Cmd Procedure SSR Delete Monitor
  • 衛星指令規格類型在沿用福衛五號的 TTQ/MPQ load 外，新增 GEO Command。
  • 資料庫必須保留衛星記憶體參數、電力參數、所有活動資料與軌道事件等。
  • 排程系統 (Primary/Backup) 之間互為備援，資料庫需每日同步。
  • 第三方軟體限制：原福衛五號 MPSS 需限制 Chrome 版本，福衛八號 MPSS 將修正 Gantt Chart 應用程式取代 Chrome 版本限制。
  • Memory Usage：系統無限制，但建議 32GB 以上。
3.3.4.5 操作注意事項 Operational Considerations
  • 要成功產生海外地面站接觸報告，MPSS 系統中必須先定義好海外站資訊（資源配置 CSU），且 FDF 產生的軌道事件也必須包含該相關地面站，並在所需時間區段內將軌道事件數據匯入到 MPSS 數據庫中（軌道事件 CSC）。
  • Product Generation CSU 需能夠訪問操控平台 XPSOC 指令資料庫，根據排程結果搭配 event template 以便可產生衛星指令集。

4.3

MCC Detail Design

Detailed Component Design
CSC 1 Orbit Event Process CSC  ·  軌道事件處理

Orbit Event Process CSC 主要是根據 FDF 次系統所提供的衛星軌道事件資料，產生與衛星相關之地面站時間、角度等資訊，以作為排程的依據。

Design Specification / Constraints
  • MCC 將使用從 FDF 所提供的資料產生酬載與衛星活動規劃與排程。
  • FDF 將提供 2 種檔案，包含軌道事件結果 (Rev Report, Contact Report)。
  • 軌道事件處理功能將使用「唯一標識符」來識別並解析軌道資料並匯入數據庫中。
  • MPSS 應提供介面透過 GCN 收集所有 FDF 信息或數據，包括匯入軌道事件檔案。
  • MPSS 必須能夠支援衛星軌道轉換操作。
軌道事件種類 (Orbit Events)
  • Ascending Node / Descending Node
  • Orbit Number
  • Earth/Sun Penumbra/Umbra Enter/Exit
  • Contact 0° rise/fade of S-Band Sites
  • Contact 10° rise/fade of S-Band Sites
  • Contact 11.5° rise/fade of X-Band Sites
  • Contact 30° rise/fade of X-Band Transportable Sites
  • Maximum elevation of contact
Inputs
  • Orbit Events (from FDF)
  • Orbit Maneuver Plan
Outputs (MPSS DB Tables)
  • contact
  • eclipse
  • rev
CSC 2 Resource Management CSC  ·  資源管理

Resource Management CSC 負責定義衛星與地面站之資源參數與相關時間參數，並依衛星規格訂定限制參數。細分為 3 個 CSU。

CSU 1：Resource Configuration CSU

用於管理調度程序使用的地面站和衛星資源，提供 GUI 操作介面，讓任務分析師可有彈性地定義/修改/刪除電源管理、衛星記憶體配置、地面站維護週期與海外地面站可用性等。

Inputs

GUI 操作介面，可彈性新增或修改地面站及衛星資源

Outputs (MPSS DB Tables)
  • dutycycle
  • recorder
  • recorderps
  • site
  • siteps
  • satellite
  • satelliteps
CSU 2：RTS Schedule Management CSU

向海外地面站進行排程請求，並確認所請求的時間區段可順利支援任務。發送海外站使用請求，等海外站評估後回傳允許使用報告。請求/確認格式參考 Remote Tracking Station ICD (FS8-ICD-0003)。

Inputs

GUI 操作介面，輸入欲進行排程時間區段

Outputs

產出符合海外地面站請求格式檔案（格式參考 FS8-ICD-0003）

CSU 3：RTCS Management CSU

RTCS (Relative Time Command Sequence) Management 用來管理事件模板 (Event Template)，包含執行每種活動類型所需的命令序列。活動對應的事件模型以 ASCII 格式儲存在 MPSS 系統內。

支援的 Event Template 類型
  • Contact：ACQ_S.etf、ACQ_E.etf
  • Science Payload PBK：Science_PLD_PBK.etf
  • Cmd Proc：RTCMD or MISC or Science_Pld_PROC
  • Maneuver Event：ATT_MAN.etf
  • Orbit Maneuver：ORB_MAN.etf
  • RSI Playback：RSI_PBK.etf
  • RSI Record：RSI_REC.etf
  • SSR Delete：SSR_DEL.etf
  • Sup Maneuver：GOHOME_MAN.etf
CSC 3 Scheduling / Rescheduling CSC  ·  排程事件規劃

排程核心部份，提供預先排程、特定酬載排程、手動排程調整與排程檢驗等功能，並依排程結果產生相對應的報表供操作人員參考。細分為 7 個 CSU。

CSU 1：Session Manager CSU

定義所要進行排程的時間區段，並引入相關的衛星及地面站資源組態。

CSU 2：Activity Request Editor CSU

提供介面供使用者手動建立新的衛星活動。福衛八號目前支援的衛星活動類型如下：

  • 健康狀態觀測
  • 衛星轉動
  • RSI 取像
  • RSI 下載
  • 科學資料下載
  • 軌道傳換
  • 衛星通訊開關
CSU 3：Cold Scheduler CSU

進行預先排程，針對既有的衛星及地面站資源組態進行確認。排程過程必須符合操作規則與限制條件（福衛八號規則文件 [TBD]）。繼承上個 session 遺留下來的活動並產生本 session 所有健康狀態觀測活動。

CSU 4：External Activity Scheduler CSU

因應任務酬載 (RSI 或 Science Payload) 需求進行特定酬載的排程作業。RSI 與酬載外部請求檔案格式必須符合 Ground Segments ICD (FS8-ICD-0001)。

Inputs

RSI 與 Science Payload Mission Timeline

Outputs
  • RSI Record
  • RSI Playback
  • SSR Delete
  • Maneuver
CSU 5：Gantt Chart CSU

提供圖形化介面呈現特定時段排程結果，包含可調度衛星、地面站、軌道事件、衛星活動等。讓任務分析師 (MA) 可以手動調整優化排程結果。

CSU 6：Scheduler Validation CSU

確認排程結果無誤且不違反既定操作規則與資源限制。驗證規則包括：

  • RSI cannot overlap PBK
  • Site Turnaround time
  • Antenna unavailable for time range
  • Contact Bounds
  • Duty Cycle
  • Latitude Time Maneuver / Latitude Time Go Maneuver
  • Maneuver Interval
  • Rec RSI Capacity
  • RSI Record activity scheduled outside RFPA window
CSU 7：Schedule Report CSU

產生排程結果相關報表。報告格式必須符合 Ground Segments ICD (FS8-ICD-0001)。

Report Outputs
  • Pass Plan
  • ASR (Acquisition Schedule Report)
  • Duty Cycle Report
  • SSR Usage Report
CSC 4 Product Generation CSC  ·  衛星指令集產生

依據排程結果產生正確的衛星指令，由操作人員上傳給衛星據以執行排定的衛星活動。根據驗證過的排程結果產生 TTQ 與 MPQ 指令集。

Design Specification / Constraints
  • 需預先由 RTCS 管理模組建立每個活動所對應的事件模型。
  • 指令類型除原有的 TTQ、MPQ 外新增了 GEO command，格式定義在 CAD (Command Allocation Document) 文件中。
Inputs
  • Scheduled Events
  • Event Templates
  • Stored Command Procedures
  • MPSS Database
Outputs
  • Command Loads (TTQ / MPQ / GEO)
  • Command Reports
  • Pass Plan
CSC 5 Utilities CSC  ·  輔助工具

提供其他功能輔助排程系統作業之進行。包含 4 個 CSU。

CSU 1：Data Transfer CSU

提供 FTP 功能下載軌道資料及 RSI、科學酬載 Mission Timeline。Json file 檔定義相關 FTP 設定，User 透過 web 介面操作 FTP Process 進行檔案傳輸。

CSU 2：Product Transfer CSU

提供 FTP 功能以傳送衛星指令到適當的工作站。可透過 FTP 功能將排程後產生的衛星指令集送至 XPSOC 操作系統等待上傳給衛星。可根據時間區段與檔案類型過濾出對應的資料上傳給 XPSOC 操作系統。

CSU 3：Purge Database/File CSU

避免系統因過期資料太多導致效能低下，定期清除資料庫中特定資料表或選定區段的暫存資料。

⚠ 由 Activity Request Editor CSU 產生的衛星活動不可被清除。為維持排程的連續性，必須至少保留一段完整的時間區段資料。
CSU 4：Database Management CSU

用於管理資料庫避免資料遺失，包含資料備份與還原功能。資料庫必須能夠處理衛星記憶體參數、衛星電量消耗參數、衛星所有活動與軌道資料。透過系統定時功能，執行資料庫備份 bat file，並同步至別的主機，產生資料庫備份檔 *.dump。
